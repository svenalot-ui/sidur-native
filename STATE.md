зманим, молитвы, Теилим, брахот, компас на Иерусалим, еврейский календарь, цдака.
Дизайн: «E-ink / editorial» — тёплая бумага, чернила, золото только для действия.

## Инструменты контента

- **`tools/editor.html` + `tools/editor_server.py` + `Редактор Сидур.command`**
  Локальный WYSIWYG-редактор (двойной клик по .command). Правит `Сидур/Content/*.json`
  напрямую. Панель: Ж/К/Ч, выделение золотом, цвет; ⚡ Вставка помечает блок;
  типы блоков; порядок; живое превью. ⌘S сохраняет (делает `.bak`).
- **`tools/build_content.py`** — конвейер из docx (три версии: иврит/транслит/русский,
  роль абзаца кодируется размером шрифта: 40 часть, 34 раздел, 26 подзаголовок,
  24 текст, ≤22 ремарка):
  ```bash
  python3 tools/build_content.py --service shacharit path/to/Шахарит_три_версии.docx
  python3 tools/build_content.py --sections path/to/Разные_молитвы.docx
  ```
  Любой json с именем = id записи в liturgy.json автоматически превращает строку
  «Скоро» в полноценную службу (`LiturgyRow` сам это подхватывает, код не трогать).

## Ключевые решения

- Нусах: активен только **Эдот а-Мизрах**, остальные — заглушки «Скоро».
- **В Шаббат и Йом-Тов приложение не работает** — экран покоя от зажигания свечей
  до исхода. Даты Йом-Това из Hebcal, кэш офлайн. Уведомления в покое не приходят.
- Зманим: свой NOAA-движок + оверрайд myzmanim (ключ в `Сидур/Secrets.swift`,
  gitignored, НЕ коммитить), результат кэшируется на диск → офлайн и мгновенно.
- Календарь: встроенный набор событий 2024–2036 (`Content/calendar.json`), API
  только вне диапазона.
- Локация: без авто-GPS, выпадающий список городов сверху экрана Зманим.
- Форматирование текста: подмножество HTML (`<b><i><u><mark><span style=color>`)
  парсится `RichText.swift` в AttributedString поверх базового шрифта.
